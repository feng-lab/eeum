Mex Files:

Most of this package is written in Matlab, so should run on any system. However, a few functions also have C versions for speed. Mex files must be generated for each type of OS. Currently we have only included 64-bit Mac and Windows versions. Note that windows version mex files depends on some visual c++ 2013 dlls. You can download "vcredist_x64.exe" from "http://www.microsoft.com/en-us/download/details.aspx?id=40784" and install it if the mex files don't work properly. If you are using another OS, the package should still function fine because we have 'backup' .m files, which are called automatically if a mex version cannot be found. If you run into problems, or want increased speed, you may need to compile files manually by calling "compile.m". Before you run the mex command for the first time, you need to type "mex -setup" to tell Matlab what compiler to use. This only has to be done once per Matlab installation.

Image Stack File Formats:

Image file reading is handled by "read_stack.m". You can modify this file to support your own file format.
"V3DRAW" any size Supported (www.vaa3d.org)
"Zeiss LSM" Supported through "tiffread.m" (http://www.cytosim.org/other/)
"ClassicTiff" file <= 4GB Supported
"BigTiff" any size Supported if Matlab version is R2012b or higher
"ClassicTiff" file > 4GB Not Supported. Some software like ImageJ can generate non-standard large "ClassicTiff" file.
